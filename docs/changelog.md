# HA Dashboard — Changelog

> The single source of truth for "what shipped" in this repo. One line per merged issue, linking
> to where the full detail actually lives:
> - For a feature built via the `docs/features/<feature-slug>/` workflow, link to that slice's
>   WBS file — `/to-implementation` appends a **Delivered** section there (issue #, branch,
>   outcome) when the work lands.
> - For a small change filed straight to a GitHub issue with no WBS slice, link to the issue/PR
>   itself.
>
> Example: `- 2026-07-23 — #1 Slice 1: Repo & infra setup — [details](features/ha-dashboard/WBS/slice-1-repo-and-infra-setup.md)`
